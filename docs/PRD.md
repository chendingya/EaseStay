# 易宿酒店预订平台 - 需求文档（PRD）

## 1. 项目概述
- 项目名称：易宿酒店预订平台
- 目标：搭建连接商户与用户的酒店预订系统，商户在 PC 端发布房源，管理员审核；用户在移动端搜索、筛选并查看房源

## 2. 用户角色与权限
- C 端用户（User）：移动端查询、筛选、浏览酒店与房型
- 商户（Merchant）：PC 端录入/修改酒店信息，提交审核
- 管理员（Admin）：PC 端审核酒店上架、下架管理

## 3. 业务范围与边界
- 覆盖：酒店信息管理、审核流程、酒店查询与展示、订单创建与统计、消息通知、预设数据与申请流程
- 不覆盖：支付、订单履约、会员体系、点评系统
### 3.1 酒店信息数据维度
- 必须维度：酒店名（中/英显示）、酒店地址、酒店星级、酒店房型、酒店价格、酒店库存、酒店开业时间
- 可选维度：酒店设施、图片、优惠信息、酒店附近的热门景点/交通/商场、房型折扣与配额、房型可住人数、床宽、面积、层高、WiFi、早餐

## 4. 移动端功能需求（用户侧）
### 4.1 酒店查询页（首页）
- P0 顶部 Banner：轮播广告，点击跳转对应酒店详情页
- P0 核心查询：城市展示、关键字搜索、入住/离店日期选择、快捷标签
- P0 查询按钮：跳转列表页并传递查询参数
- P1 热门推荐：展示热门酒店卡片列表
- P0 日历组件：支持跨月选择

### 4.2 酒店列表页
- P0 顶部核心筛选：城市、入住/离店日期、间夜数、关键词展示
- P0 详细筛选区域：星级/价格/标签等维度，支持横向快捷筛选
- P0 排序：推荐/价格/星级
- P0 列表展示：酒店卡片（图、名（中/英）、星级、最低价、位置、开业时间）
- P0 上滑自动加载（Infinite Scroll）

### 4.3 酒店详情页
- P0 顶部导航栏：酒店名与返回列表页功能，支持透明渐变效果
- P0 基本信息：顶部大图轮播、酒店名（中/英）、星级、地址、设施、开业时间
- P0 日历 Banner：展示当前选择的入离时间和间夜数
- P0 房型价格列表：按价格从低到高展示
- P0 预订：房型下单创建订单
- P1 优惠信息：节日折扣/套餐等优惠展示
- P1 周边信息：热门景点/交通/商场信息展示
- P1 收藏与分享：基础交互提示

### 4.4 个人中心
- P0 登录/注册：手机号验证码注册/登录（角色固定为 User）
- P0 订单列表：查看历史订单状态
- P1 个人信息：头像、昵称修改

## 5. PC 端管理系统需求（商户/管理员）
### 5.1 登录/注册页（通用）
- P0 注册：选择角色（商户/管理员），验证码校验与密码加密存储
- P0 登录：输入账号密码，自动判断角色跳转不同首页

### 5.2 酒店录入/编辑（商户）
- P0 信息管理：酒店名（中/英）、图片、星级、设施、房型、价格、开业时间
- P0 房型扩展：可住人数、床宽、面积、层高、WiFi、早餐
- P0 地图选址：城市/地址支持地图直接选取或搜索输入
- P0 日期选择：开业时间使用日期选择器
- P0 图片上传：支持多张图片上传、预览、删除
- P1 扩展信息：周边景点/交通/商场（支持地图对接）、优惠信息
- P0 状态流转：保存后为"待审核"，数据需实时同步到 C 端

### 5.3 预设与申请系统
- P0 预设设施：系统预设常用设施标签（免费WiFi、停车场等）供商户选择
- P0 预设房型：系统预设常用房型模板供商户快速添加
- P0 预设优惠：系统预设优惠类型（早鸟优惠、周末特惠等）供选择
- P1 申请新增：商户可申请新增自定义设施/房型/优惠，需管理员审核
- P1 审核通知：申请审核通过/拒绝后，系统自动发送消息通知商户

### 5.4 工作台（商户/管理员）
- P0 新增酒店：快捷入口创建新酒店
- P0 数据统计：待审核/已上架/已下架/总数统计
- P1 批量折扣：可选择房型、设置数量与折扣力度（无需优惠名称）
- P1 批量房型操作：显示已用/空闲房间，支持下架房型与数量或调整库存
- P1 酒店列表：展示最近酒店，支持快速编辑
- P1 商户总览：统计商户名下酒店总数与月度收入

### 5.5 酒店审核列表（管理员）
- P0 酒店审核：通过/不通过，不通过需填写原因
- P1 申请审核：审核商户提交的新增设施/房型/优惠申请
- P0 上下架：已发布酒店可下线，支持恢复；下线为状态变更

### 5.6 商户酒店详情（商户）
- P0 房间总览：展示总房间、已用、空闲、下架数量与占比
- P0 订单展示：订单列表分页展示
- P0 订单统计：订单量、收入、状态分布、月度收入
### 5.7 消息中心
- P0 消息列表：查看通知、筛选未读、标记已读/全部已读
### 5.8 账户与商户管理（管理员）
- P0 商户管理：商户列表、详情、重置密码
- P0 账户管理：查看账户信息与修改密码

### 5.9 管理员酒店详情（管理员）
- P0 全局试图：具备商户端同等的数据视图权限
- P0 房间总览：展示该酒店的房间使用情况
- P0 订单展示：查看该酒店的历史订单
- P0 订单统计：查看该酒店的经营数据
- P0 审核管理：保留原有的审核、下架、恢复上架功能

## 6. 关键业务规则
- 酒店状态：pending / approved / rejected / offline
- 商户修改酒店后状态回到 pending
- 只有 approved 酒店可在 C 端展示
- 订单创建后占用房型库存，库存不足时不可下单

## 7. 交互与性能要求
- 搜索结果支持分页与无限滚动
- 详情页房型价格升序展示
- 列表页支持长列表性能优化（可选）
