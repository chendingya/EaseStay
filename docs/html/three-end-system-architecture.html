三端系统总架构图

简版视图：三端入口、服务层、数据层与部署关系

01 终端入口层

C-End 用户端

面向用户，覆盖搜索、下单和订单管理。

mobile/src/app.config.js
搜索 预订 订单

Merchant-End 商户端

面向商户，负责酒店与房型维护。

admin/src/App.jsx
酒店管理 房型管理 申请提交

Admin-End 平台管理端

面向管理员，负责审核、商户管理与运营控制。

server/src/middleware/auth.js
审核 商户管理 RBAC

商户端与管理员端共用同一前端工程，通过角色权限进行菜单与路由分流。

HTTPS / REST + JWT

02 接入与安全层

API Gateway Express

统一接口入口，负责请求接入与服务编排。

server/src/app.js / server/src/routes/index.js

Auth + RBAC 安全域

提供 JWT 鉴权与角色权限控制。

authRequired + requireRole
Domain Services / Route Aggregation

03 业务服务层

Auth & User

账号登录、用户资料与个人业务入口。

routes/auth.js + routes/user.js

Hotel & Room

酒店与房型的查询、管理与状态流转。

routes/publicHotels.js / merchantHotels.js / adminHotels.js

Request & Preset

预设能力与申请审核流程中心。

routes/requests.js + adminRequests.js + presets.js

Order & Availability

订单生命周期与库存可用性控制。

services/roomAvailabilityService.js + pricingService.js

Notifications

消息通知、未读计数与已读同步。

routes/notifications.js + notificationService.js

Map & Image Proxy

地图检索与图片代理支撑。

routes/map.js + routes/image.js
PostgreSQL / External API / CDN

04 数据与外部依赖层

Supabase PostgreSQL

核心业务数据存储。

server/src/config/supabase.js

AMap Service

提供地理编码与位置检索。

AMAP_KEY + mapService.js

Image Sources

图片资源与访问安全控制。

IMAGE_PROXY_ALLOW_HOSTS
