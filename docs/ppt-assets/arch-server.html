服务端分层架构（Express + Supabase）

Router Layer

统一装配业务路由与中间件

Controller Layer

参数解析 + 权限守卫 + 响应

Service Layer

状态机/库存/订单/通知规则

Data Layer

PostgreSQL + Map

鉴权与权限（中间件）

authRequired：校验 Bearer Token，失败返回 401

requireRole(role)：校验角色，失败返回 403

在路由注册阶段统一应用，前后端权限语义一致

状态机与一致性

订单：pending_payment -> confirmed -> finished / cancelled

酒店：pending -> approved / rejected / offline

关键规则统一沉淀 service，避免前端分散判定

router.use('/admin/hotels', authRequired, requireRole('admin'), adminHotelRoutes) router.use('/merchant/hotels', authRequired, requireRole('merchant'), merchantHotelRoutes) router.use('/requests', authRequired, requestRoutes)
