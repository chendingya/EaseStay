# 移动端 H5 本地联调（手机访问）

## 架构说明

```
手机浏览器
    │  访问 http://<电脑IP>:10086
    ▼
Taro H5 开发服务（端口 10086）
    │  代理 /api  /health
    ▼
后端服务（127.0.0.1:4100）
```

`config/dev.js` 已将 `/api` 和 `/health` 请求代理到本机后端，**手机无需直接访问 4100 端口，也不需要修改任何接口地址配置**。

---

## 1. 启动后端服务

在 `server` 目录下执行：

```bash
cd server
npm run start
```

确认后端在 `127.0.0.1:4100` 已正常运行。

## 2. 启动 H5 开发服务

新开一个终端，在 `mobile` 目录下执行：

```bash
cd mobile
npm install   # 首次或依赖变更时执行
npm run dev:h5
```

启动成功后会看到：

```
Local:   http://localhost:10086
Network: http://<电脑局域网IP>:10086
```

开发服务在 `config/dev.js` 中配置为：

- **Host**：`0.0.0.0`（监听所有网卡，允许局域网访问）
- **Port**：`10086`
- **Proxy**：`/api`、`/health` → `http://127.0.0.1:4100`

## 3. 获取电脑局域网 IP

**Windows**：

```powershell
ipconfig | Select-String "IPv4"
```

**Mac / Linux**：

```bash
ifconfig | grep "inet "
```

取连接 Wi-Fi 那块网卡的地址（通常形如 `192.168.x.x`），**不要用** `172.x.x.x`（虚拟网卡）或 `169.x.x.x`（未分配）。

## 4. 手机访问

确保手机和电脑连接**同一 Wi-Fi**，在手机浏览器访问：

```
http://<电脑局域网IP>:10086
```

示例：

```
http://192.168.31.157:10086
```

---

## 5. 常见问题排查

### 手机访问超时 / 无法连接

**检查防火墙**（Windows，需管理员权限）：

```powershell
netsh advfirewall firewall add rule name="Taro H5 10086" dir=in action=allow protocol=TCP localport=10086
```

**其他常见原因**：

| 现象 | 原因 | 解决方式 |
|---|---|---|
| 手机显示"无法访问" | 防火墙拦截 10086 端口 | 执行上方防火墙命令 |
| 页面加载但接口 404 | 后端未启动 | 先启动 `server` |
| 页面加载但接口 502 | 后端崩溃或端口不对 | 检查后端是否跑在 4100 |
| 页面加载但接口 504（如 `/api/auth/sms/send`） | H5 devServer 代理超时（10086→4100）或后端瞬时卡顿 | 先确认 `http://127.0.0.1:4100/health` 返回 200；若正常，重启 `mobile` 的 `npm run dev:h5` 后重试 |
| 连接成功但很慢 | 开了 VPN / 手机代理 | 关闭手机的 VPN 和代理 |
| 无法找到电脑 IP | 手机和电脑在不同 Wi-Fi | 确认连接同一局域网 |

### 接口地址无需手动配置

> ⚠️ **注意**：无需在 `.env.local` 中设置 `TARO_APP_API_BASE` 为电脑 IP。
> `config/dev.js` 中的 Webpack DevServer Proxy 已在服务端完成转发，手机只需访问 10086 端口即可。

> 登录/注册交互与认证行为说明，请查看 `docs/PRD.md` 与 `docs/SDD.md` 对应章节。
